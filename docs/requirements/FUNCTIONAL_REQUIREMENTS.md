# LifeGrid — Solution Bootstrapping Requirements

## Context
This document captures the finalized, clarified requirements for the initial physical scaffolding of the LifeGrid .NET 10 solution. It is the input artifact for the implementation plan at `docs/plans/bootstrap-solution-scaffolding.md`.

---

## 1. Solution File
- A single `LifeGrid.slnx` file at the repository root (`.slnx` is the new XML-based solution format generated by default in .NET 10 SDK — legacy `.sln` is not created).
- All projects must be registered in this solution file.

---

## 2. Directory Layout
All source projects must live under a `src/` subdirectory.  
All test projects must live under a `tests/` subdirectory.

```
c:\Code\LifeGrid\
├── LifeGrid.slnx
├── src\
│   ├── LifeGrid.Domain\
│   ├── LifeGrid.Application\
│   ├── LifeGrid.Infrastructure\
│   └── LifeGrid.Presentation\
└── tests\
    ├── LifeGrid.Domain.Tests\
    ├── LifeGrid.Application.Tests\
    └── LifeGrid.Infrastructure.Tests\
```

## 4. Test Project Specifications
All test projects use: `xunit` (bundled with template), `FluentAssertions 8.10.0`, `NSubstitute 5.3.0`.

| Project | Template | References | Extra Package |
|---|---|---|---|
| `LifeGrid.Domain.Tests` | `xunit net10.0` | `LifeGrid.Domain` | — |
| `LifeGrid.Application.Tests` | `xunit net10.0` | `LifeGrid.Application`, `LifeGrid.Domain` | `MediatR 14.1.0` |
| `LifeGrid.Infrastructure.Tests` | `xunit net10.0` | `LifeGrid.Infrastructure`, `LifeGrid.Application` | `Microsoft.EntityFrameworkCore.InMemory 10.0.9` |

---

## 5. Dependency Rules (Clean Architecture Inward-Only)
```
Presentation  ──►  Application  ──►  Domain
Infrastructure ──►  Application  ──►  Domain
```
- **Forbidden cross-references:** Presentation must never reference Infrastructure or Domain directly. Infrastructure must never reference Presentation.

---

## 6. Acceptance Criteria
- `dotnet build LifeGrid.slnx` at the solution root must exit with code 0 (zero errors, zero warnings from P2P misconfiguration).
- All seven projects must appear in `dotnet sln list`.
- The folder tree must be reported upon successful build.
